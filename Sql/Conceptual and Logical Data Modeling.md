# Logical Model Readiness Check
- All business  rules are captured
- Data types and constraints are selected 
- keys and relationships are defined

## Within physical models, you will define:
- Indexing strategies
- Partitioning
- Storage formats
- Platform/vendor specific features

# Summary
- Conceptual model describe Business  Concept  and Relationship 
-  Logical Models Define Tables, Columns, Keys, relationships an business rules
- Model Review and documentation prevent ambiguity and rework
- A clear logical model is key  in supporting  the development of a physical model
